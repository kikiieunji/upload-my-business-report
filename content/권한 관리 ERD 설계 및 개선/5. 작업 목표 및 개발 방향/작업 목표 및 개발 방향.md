
### 1. 개발 방향

- **Response 값 기반으로 DDD 설계 방향 수립**
    
- **비즈니스 로직은 변경하지 않고**,  
    **AOP + 커스텀 어노테이션** 방식으로 권한 및 `user_scope` 처리 가능 여부 검증
    

---

### 2. API 및 문서화

- **API 명세 수정 사항 문서화**
    
    - 정리 후 **민택님께 전달**
        
- **Payment Controller Swagger**를 기준으로 구현 및 검증
    
- **현재 존재하는 DTO 전체 활용**하여 개발 진행
    

---

### 3. 기능 범위

- 핵심 기능:
    
    - **조회**
        
    - **전자결재**
        
- 위 두 영역을 **메인 대상으로 우선 적용**
    

---

### 4. 개발 및 리뷰 방식

- **API 단위로 작업 후 Pull Request 등록**
    
    - API 하나씩 올려도 무관
        
    - **Swagger 테스트까지 완료된 상태로 PR**
        
- PR 기반으로 코드 및 설계 리뷰 진행
