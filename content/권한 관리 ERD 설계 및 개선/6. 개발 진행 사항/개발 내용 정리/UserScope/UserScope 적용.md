- @ScopeCheck를 어떻게 가지고 갈 것인가?
	- interceptor방식과 AOP 방식 비교

| 구분                    | 공통점                    | Interceptor                                     | AOP                                       |
| --------------------- | ---------------------- | ----------------------------------------------- | ----------------------------------------- |
| 목적                    | 공통 로직(권한/로깅/검증) 중복 제거  | 컨트롤러 진입 전/후 HTTP 요청 흐름 제어                       | 메서드 실행 전/후/예외 시점에 횡단 관심사 적용               |
| 적용 위치                 | 둘 다 비즈니스 코드 바깥으로 분리 가능 | Spring MVC 계층 (웹 요청 처리 파이프라인)                   | Spring Bean 메서드 계층 (Controller/Service 등) |
| 트리거 기준                | 조건에 따라 실행 가능           | URL, HandlerMethod, HTTP 요청 정보 기반               | 어노테이션, 패키지, 메서드 시그니처(Pointcut) 기반         |
| HTTP 정보 접근            | 가능                     | 매우 쉬움 (HttpServletRequest, param, header 직접 접근) | 가능하지만 상대적으로 우회적 (JoinPoint + 추가 처리)       |
| @RequestBody 접근 가능 여부 | 케이스별                   | 기본 preHandle에서는 바디 접근 어려움 (별도 래핑 필요)            | 메서드 인자 그대로 접근 쉬움                          |
| 디버깅 난이도               | 공통 로직이라 추적 필요          | 비교적 쉬움 (요청 흐름에서 눈에 보임)                          | 상대적으로 어려움 (프록시/포인트컷 이해 필요)                |
| 적용 범위                 | 재사용 가능                 | 웹 요청에 특화                                        | 웹 외에도 Service, Scheduler 등 전 계층 적용 가능     |
| 실수 방지                 | 누락 줄임                  | 인터셉터 등록된 라우트는 일관 처리                             | 포인트컷 미스매치 시 누락 가능                         |
| 성능 체감                 | 보통 미미                  | 요청 단위 고정 오버헤드                                   | 프록시 + 포인트컷 평가 오버헤드 (보통 미미)                |
| 현재 케이스 적합도            | 둘 다 가능                 | @RequestParam 기반 Scope 체크에 매우 적합                | 복잡한 인자 구조/다계층 공통 정책이면 유리                  |
- Interceptor의 한계 : HttpServletRequest 수준에서 동작하기 때문에 companyId같은 파라미터를 꺼내는것이 까다롭다 -> POST 요청의 JSON 바디는 한 번 읽으면 사라지는 스트림이라 별도 처리 필요 
- AOP의 장점 : AOP는 이미 파라미터가 다 해석된 Java 시호출 시점에 개입하므로 구현이 훨씬 직관적이고 간단함

-> Scope는 컨트롤러 진입 시점의 입구 차단 :: 권한검증과 똑같이 @AOP 적용
